# Retail Sales & Customer Segmentation (RFM Analysis)

## 📌 Project Overview
This repository contains the technical implementation of a behavioral customer segmentation model using the **RFM (Recency, Frequency, Monetary)** framework. The project processes a large-scale retail dataset to identify high-value customers, detect churn risks, and provide data-driven foundations for targeted marketing strategies.

## ⚙️ Technical Workflow & Data Pipeline

### 1. Data Cleansing
The raw dataset consisted of **1,067,371 rows**. The preprocessing pipeline executed the following operations to ensure data integrity:
* **Missing Value Handling:** Dropped records with null `Customer ID`s to ensure accurate user-level aggregation.
* **Anomaly Removal:** Filtered out canceled transactions (Invoice numbers starting with 'C') and invalid numerical entries (`Quantity` <= 0, `Price` <= 0).
* **Feature Engineering:** Created a `TotalSales` feature by multiplying `Quantity` and `UnitPrice`.
* **Result:** A clean, production-ready dataset of **805,549 rows**.

### 2. RFM Modeling
Aggregated the transactional data into a customer-centric view for **5,878 unique users**:
* **Recency:** Days elapsed since the customer's last purchase relative to a reference date.
* **Frequency:** Count of unique invoices per customer.
* **Monetary:** Sum of total revenue generated by the customer.
Used `pd.qcut()` to assign quartile scores (1-4) for each metric, which were then concatenated to form the final `RFM_Score`.

### 3. Segmentation Logic
Customers were classified into actionable segments based on their RFM scores:
* **Best Customers:** Highest scores across all metrics (e.g., '444').
* **Loyal Customers:** High frequency and recent activity.
* **At Risk:** High historical frequency but low recent activity.
* **Lost:** Low scores across all dimensions.
